室内定位sdk描述

室内定位sdk

需求概述

  • 数据独立
    • 概述
      • 每个商圈/区域所对应的数据为一个单位
      • app需要向sdk注册自己使用的数据单位（预置的数据单位需要声明）
      • app可以预置数据或运行后从云端下载，或采用混合方案
      • app需要告诉sdk当前所在商圈/区域
    • 数据升级方案
      • 制定数据升级规范
      • sdk内部提供自带的升级通道升级数据（墨兔维护）
      • 允许自定义升级文件服务器（制定升级文件规范提供给第三方）
      • 允许第三方自定义具体的升级方案参数，比如：检查周期、升级是否wifi条件、云端主动push更新等
  • 模块独立
    • 现在模块中引用的mapbox的代码要替换，如：地理坐标转平面坐标等
    • 依赖注入框架/响应编程框架等移除（以后墨兔的开放支持组件均不引用此类库）
  • 其他
    • sdk混淆（android）
    • 上传jcenter（android）

总体设计

数据库设计

表名 说明
mall 保存商圈/区域信息
block 一个商圈的不同区域，如：海岸城东座、西座
bloor 商场中一个区域
floor 楼层信息
beacon iBeacon设备信息
connect_bloor bloor的连通表
config 保存数据版本信息（单独数据库保存）

具体表的详细设计请参考数据库设计说明

接口设计

  1. 注册数据单元
  2. 设置本地数据库地址
  3. 设置当前所在商圈
  4. 设置开关数据库自动升级
  5. 配置自己的数据升级地址
  6. 设置自动更新相关参数
  7. 位置监听接口，实时返回当前位置信息（坐标、楼层、bloor等等）
  8. 设置前后台扫描频率周期
  9. 开启关闭扫描
  10. 设置beacon UUID
